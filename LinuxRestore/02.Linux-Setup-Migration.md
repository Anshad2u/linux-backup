### 0.2 Stop Services

### Stop guacd service
Command: `sudo systemctl stop guacd`
Purpose: Stop Guacamole proxy daemon
Service: guacd (port 4822)
Note: Used by Guacamole to proxy RDP/VNC connections

### Stop guacd completely and disable
Command: `sudo systemctl stop guacd && sudo systemctl disable guacd`
Purpose: Stop and disable guacd service permanently

### Stop manually installed Tomcat 9
Command: `/opt/tomcat9/bin/shutdown.sh`
Purpose: Stop manually installed Tomcat 9 (Guacamole webapp)

### Check for running Java processes from custom Tomcat
Command: `ps aux | grep -E 'guacd|tomcat|java' | grep -v grep`
Purpose: Verify no Guacamole/Tomcat Java processes remain
Note: Found multiple Java processes - need to kill manually

### Kill remaining Guacamole Java processes
Command: `pkill -f 'guacamole' && pkill -f 'tomcat9'`
Purpose: Kill any remaining Guacamole/Tomcat processes
Note: guacd runs as root, other Tomcat processes as user 'anshad'

### Force kill processes if needed
Command: `kill -9 <PID1> <PID2> ...`
Purpose: Force kill stubborn processes

Note: After cleanup, these commands should be run to fully remove packages:
```bash
sudo systemctl stop guacd
sudo systemctl disable guacd
sudo apt-get purge --remove guacd tomcat10 tomcat10-admin tomcat10-common libtomcat10-java
sudo rm -rf /etc/guacamole
sudo rm -rf /var/lib/tomcat10
sudo rm -rf /opt/tomcat9
sudo apt-get autoremove
```

### 2.3 Install Docker + Docker Compose

**IMPORTANT**: Docker installation requires sudo. Run manually with sudo password:

```bash
# Install Docker
sudo apt-get update
sudo apt-get install -y docker.io docker-compose-plugin

# Enable and start Docker
sudo systemctl enable docker
sudo systemctl start docker

# Add user to docker group (log out/in required after)
sudo usermod -aG docker $USER
```

#### Verify Docker installation
Command: `docker --version && docker compose version`
Purpose: Verify Docker is working
Expected: Docker 24.x+ and Docker Compose 2.x+

#### Test Docker
Command: `docker run --rm hello-world`
Purpose: Test Docker container capability

---

### 2.4 Install Coolify

**IMPORTANT**: Coolify installation requires sudo. Run manually:

