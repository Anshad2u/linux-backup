# Guacamole Installation Guide

This guide covers setting up Guacamole (remote desktop gateway) with XRDP on Ubuntu/Linux and exposing it via Cloudflare Tunnel.

## System Requirements

- Ubuntu 22.04+ or similar Linux distribution
- Tomcat 9 (for Guacamole webapp)
- guacd (Guacamole proxy daemon)
- XRDP (for remote desktop access)
- Cloudflare account with domain

## Installation Steps

### 1. Install guacd (Proxy Daemon)

```bash
sudo apt-get update
sudo apt-get install -y guacd libjakarta-servlet-api-java
```

### 2. Install Guacamole Webapp for Tomcat 9

Download the official Guacamole WAR file (use 1.5.x for Tomcat 9):

```bash
wget https://downloads.apache.org/guacamole/1.5.5/binary/guacamole-1.5.5.war
sudo cp guacamole-1.5.5.war /var/lib/tomcat9/webapps/guacamole.war
sudo systemctl restart tomcat9
```

**Important:** Use Guacamole 1.5.x for Tomcat 9 (javax.servlet). Guacamole 1.6+ requires Tomcat 10+ (jakarta.servlet).

### 3. Configure Guacamole

Edit the Guacamole properties file:

```bash
sudo nano /etc/guacamole/guacamole.properties
```

Add:
```
guacd-hostname: localhost
guacd-port: 4822
```

Restart Tomcat:
```bash
/opt/tomcat9/bin/shutdown.sh
/opt/tomcat9/bin/startup.sh
```

### 4. Install and Configure XRDP

```bash
sudo apt-get install -y xrdp xorgxrdp
sudo systemctl enable xrdp
sudo systemctl start xrdp
```

### 5. Install Tomcat 9 (if not already installed)

```bash
cd /tmp
wget https://archive.apache.org/dist/tomcat/tomcat-9/v9.0.102/bin/apache-tomcat-9.0.102.tar.gz
tar -xzf apache-tomcat-9.0.102.tar.gz
sudo cp -r apache-tomcat-9.0.102 /opt/tomcat9
sudo chown -R $USER:$USER /opt/tomcat9
```

### 6. Set Up Cloudflare Tunnel

#### Install cloudflared:
```bash
wget https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64
sudo mv cloudflared-linux-amd64 /usr/local/bin/cloudflared
sudo chmod +x /usr/local/bin/cloudflared
```

#### Authenticate:
```bash
cloudflared tunnel login
```

#### Create a tunnel:
```bash
cloudflared tunnel create <tunnel-name>
# Example: cloudflared tunnel create guac-server
```

#### Configure ingress rules:
```bash
nano ~/.cloudflared/config.yml
```

Add:
```yaml
tunnel: <tunnel-id>
credentials-file: /home/<user>/.cloudflared/<tunnel-id>.json

ingress:
  - hostname: your-domain.com
    service: http://localhost:8080
  - service: http_status:404
```

#### Add DNS route:
```bash
cloudflared tunnel route dns add your-domain.com
```

#### Start the tunnel:
```bash
cloudflared --config ~/.cloudflared/config.yml tunnel run
```

### 7. Configure Cloudflare DNS

If using the Cloudflare Dashboard:

1. Login to https://dash.cloudflare.com
2. Select your domain > DNS
3. Delete existing A/AAAA records for the subdomain
4. Add CNAME record:
   - Type: CNAME
   - Name: subdomain (e.g., guac)
   - Target: `<tunnel-id>.cfargotunnel.com`
   - Proxy: ON (orange cloud)

Or use API:
```bash
curl -X POST "https://api.cloudflare.com/client/v4/zones/<zone-id>/dns_records" \
  -H "Authorization: Bearer <api-token>" \
  -H "Content-Type: application/json" \
  -d '{
    "type":"CNAME",
    "name":"subdomain.domain.com",
    "content":"<tunnel-id>.cfargotunnel.com",
    "proxied":true
  }'
```

### 8. Default Credentials

Default Guacamole login (from database schema):
- Username: `guacadmin`
- Password: `guacadmin`

The default password "guacadmin" comes hashed in the schema/002-create-admin-user.sql file.

### 9. Database Authentication (MySQL)

For production, configure MySQL database authentication:

1. Install MySQL:
```bash
sudo apt-get install -y default-mysql-server default-mysql-client
```

2. Create database and user:
```bash
sudo mysql -u root -e "CREATE DATABASE guacdb;"
sudo mysql -u root -e "CREATE USER 'guacadmin'@'localhost' IDENTIFIED BY 'Nopassword.123';"
sudo mysql -u root -e "GRANT ALL PRIVILEGES ON guacdb.* TO 'guacadmin'@'localhost'; FLUSH PRIVILEGES;"
```

3. Download JDBC extension:
```bash
wget https://downloads.apache.org/guacamole/1.6.0/binary/guacamole-auth-jdbc-1.6.0.tar.gz
tar -xzf guacamole-auth-jdbc-1.6.0.tar.gz
sudo cp guacamole-auth-jdbc-1.6.0/mysql/*.jar /etc/guacamole/extensions/
```

4. Get MySQL connector:
```bash
wget https://repo1.maven.org/maven2/com/mysql/mysql-connector-j/8.0.33/mysql-connector-j-8.0.33.jar
sudo cp mysql-connector-j-8.0.33.jar /etc/guacamole/lib/
```

5. Configure guacamole.properties:
```
guacd-hostname: localhost
guacd-port: 4822
mysql-hostname: localhost
mysql-port: 3306
mysql-database: guacdb
mysql-username: guacadmin
mysql-password: Nopassword.123
```

6. Initialize database:
```bash
mysql -u guacadmin -pNopassword.123 guacdb < /path/to/guacamole-auth-jdbc-1.6.0/mysql/schema/001-create-schema.sql
mysql -u guacadmin -pNopassword.123 guacdb < /path/to/guacamole-auth-jdbc-1.6.0/mysql/schema/002-create-admin-user.sql
```

**Change passwords immediately in production!**

## Service Management

### Start/Stop Services

```bash
# Guacamole/Tomcat
/opt/tomcat9/bin/startup.sh
/opt/tomcat9/bin/shutdown.sh

# guacd
sudo systemctl start guacd
sudo systemctl stop guacd

# XRDP
sudo systemctl start xrdp
sudo systemctl stop xrdp

# Cloudflare Tunnel
cloudflared --config ~/.cloudflared/config.yml tunnel run
```

### Check Status

```bash
# Local access
curl -I http://localhost:8080/guacamole/

# Remote access
curl -I http://your-domain.com/guacamole/

# Services
ps aux | grep guacd
ps aux | grep xrdp
ps aux | grep cloudflared
```

### Logs

```bash
# Tomcat
tail -f /var/lib/tomcat9/logs/catalina.out

# cloudflared
tail -f ~/.cloudflared/logs
```

## Connecting

### Via Guacamole Web Interface

1. Open browser to `http://your-domain.com/guacamole/`
2. Login with credentials
3. Configure new connection:
   - Protocol: RDP
   - Hostname: localhost (or 127.0.0.1)
   - Port: 3389

### Via Direct RDP

Connect RDP client directly to `your-domain.com:3389`

## Troubleshooting

### Common Issues

1. **404 on Guacamole**: Wrong WAR version
   - Use Guacamole 1.5.x for Tomcat 9
   - Use Guacamole 1.6.x for Tomcat 10

2. **ClassNotFoundException: javax.servlet.ServletContextListener**
   - Install javax servlet compatibility: `sudo apt-get install libjakarta-servlet-api-java`
   - Or use correct Tomcat version

3. **Cloudflare 530 error**
   - Tunnel not running: `cloudflared tunnel run`
   - DNS not pointing to tunnel: Check CNAME record

4. **guacd not running**
   ```bash
   sudo systemctl start guacd
   sudo systemctl enable guacd
   ```

5. **XRDP not connecting**
   - Check port: `netstat -tlnp | grep 3389`
   - Restart: `sudo systemctl restart xrdp`

## Quick Reference

| Service | Port | Command |
